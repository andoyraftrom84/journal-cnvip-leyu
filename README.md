# journal-cnvip-leyu

## 项目简介

`journal-cnvip-leyu` 是一个用于归档和发布多个独立 HTML 页面的仓库。本仓库不针对任何特定域名或网站，主要作为静态页面的集中存储与展示空间。

## 目录说明

```
.
├── pages/          # 存放所有独立 HTML 页面文件
├── assets/         # 页面引用的资源文件（CSS、JS、图片等）
├── README.md       # 本文件
└── LICENSE         # 许可证文件（如适用）
```

- **pages/**：每个页面以独立的 `.html` 文件形式存放，文件名可反映页面内容或用途。
- **assets/**：公共资源文件按类型组织，便于页面引用和维护。

## 页面归档说明

- 所有页面均为独立静态 HTML，不依赖后端服务。
- 页面之间无强制关联，各自独立运行。
- 归档内容可能包含各类主题的页面，不限定于某一领域。

## 维护说明

- 新增页面时，请将 HTML 文件放入 `pages/` 目录，并确保引用的资源路径正确。
- 如需添加全局资源，请放入 `assets/` 对应子目录。
- 提交前建议检查页面是否能在本地正常打开。
- 本仓库不保证页面内容的持续更新或可用性。

## 使用

1. 克隆本仓库到本地。
2. 直接使用浏览器打开 `pages/` 下的 `.html` 文件即可查看对应页面。
3. 也可通过 GitHub Pages 或类似服务部署后访问。
